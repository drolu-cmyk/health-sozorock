## Feedback Categories

Use one category per feedback item.

| Category | Use when feedback is about |
| --- | --- |
| Strategic clarity | Whether the overall purpose, promise, and positioning are clear. |
| Resident clarity | Whether the resident-facing experience is simple, private, and understandable. |
| County value | Whether the county operating intelligence layer communicates value. |
| Design and usability | Layout, readability, mobile/tablet/desktop experience, labels, spacing, or navigation. |
| Trust and safety | Non-clinical boundary, consent posture, data boundary, and human review clarity. |
| Legal/compliance concern | Any legal, compliance, public claims, or review concern. |
| Technical issue | Broken page, console issue, layout defect, link issue, or preview behavior. |
| Partnership or claims concern | Any wording that could imply unsupported endorsement or formal relationship. |
| Future feature idea | Useful idea that is not needed for this preview review. |
| Out of scope | Request that belongs outside this preview share packet. |

## Severity Guide

| Severity | Meaning |
| --- | --- |
| Critical | Blocks internal review or creates a trust, safety, legal, or public-claims risk. |
| High | Should be addressed before broader stakeholder review. |
| Medium | Useful improvement before pilot planning or wider review. |
| Low | Polish, wording, or future refinement. |

## Status Guide

| Status | Meaning |
| --- | --- |
| New | Feedback has been recorded and not yet reviewed. |
| Reviewing | Owner is assessing the feedback. |
| Accepted | Feedback should become a follow-up issue or action. |
| Deferred | Feedback is useful but not needed now. |
| Out of scope | Feedback does not fit the preview share packet scope. |
| Closed | Feedback has been resolved or intentionally closed. |

## Out-Of-Scope Reminder

Do not use this log to request implementation of:

- custom domain configuration
- DNS changes
- backend services
- database
- authentication
- forms
- storage
- notifications
- email/SMS
- live maps
- live AI
- resident data capture
- PHI workflow
- clinical workflow

Those items require separate issue framing, human approval, and guardrail review before implementation.
